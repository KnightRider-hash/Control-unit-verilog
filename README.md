# RISC-V Control Unit (Verilog)  #
This repository contains a Verilog implementation of a Main Control Unit and ALU Control Unit designed for a single-cycle RISC-V processor.
The module decodes incoming instruction fields (opcode, funct3, funct7) and generates the necessary control signals to drive the processor's datapath, including the ALU, data memory, branch logic, and register file.
## Features ##
* Combinational Logic Design: Fully combinational implementation using always @(*) blocks for immediate signal generation.
* Broad Instruction Support: Decodes standard base integer instructions (RV32I/RV64I).
* Supported Instruction Formats:
  * R-Type: Standard arithmetic/logic operations.
  * I-Type: Immediate arithmetic, Loads, and jalr.
  * S-Type: Stores.
  * B-Type: Branches.
  * U-Type: Upper immediates (lui, auipc).
  * J-Type: Jumps (jal).
* Integrated ALU Control: Generates a 4-bit op code directly for the ALU based on a primary aluop stage and instruction function bits.

### Module: `Cu` (Control Unit)

| Port Name | Direction | Width | Data Type | Description |
| --- | --- | --- | --- | --- |
| `opcode` | Input | 7-bit (`[6:0]`) | `wire` | Main instruction opcode |
| `funct3` | Input | 3-bit (`[2:0]`) | `wire` | 3-bit function code |
| `funct7` | Input | 1-bit | `wire` | 1-bit function code (from `funct7` field) |
| `Rgwrite` | Output | 1-bit | `reg` | Register write enable |
| `branch` | Output | 1-bit | `reg` | Branch flag |
| `alusrc` | Output | 1-bit | `reg` | ALU source selector |
| `memread` | Output | 1-bit | `reg` | Data memory read enable |
| `memwrite` | Output | 1-bit | `reg` | Data memory write enable |
| `jump` | Output | 1-bit | `reg` | Jump flag (`jal`/`jalr`) |
| `memtreg` | Output | 2-bit (`[1:0]`) | `reg` | Memory to register multiplexer select |
| `aluop` | Output | 2-bit (`[1:0]`) | `reg` | ALU operation category |
| `immsrc` | Output | 2-bit (`[1:0]`) | `reg` | Immediate generator selector |
| `op` | Output | 4-bit (`[3:0]`) | `reg` | Specific ALU operation code |

## Architecture Overview ## 
The Cu module acts as a two-stage decoder integrated into a single file:
1. Main Control (Opcode Decoder): Uses a case(opcode) statement to evaluate the 7-bit opcode. It sets default datapath routing signals (alusrc, memtreg, etc.) and assigns a 2-bit aluop code.
2. ALU Control Decoder:
   Takes the 2-bit aluop generated by the Main Control, along with the funct3 and funct7 inputs, to determine the exact 4-bit operation (op) the ALU needs to perform.
   * aluop = 00: Force Addition (Loads/Stores)
   * aluop = 01: Force Subtraction (Branches)
   * aluop = 10: Decode via funct3 and funct7 (R-Type / I-Type ALU operations)
